- bot_core restart followed the 09:34 docs commit by ~14 min — consistent with auto-redeploy on push.

**Caveat:** other restart triggers (memory limits, Railway internal events, manual restart) cannot be ruled out without access to `list-deployments`. **Recommend opening this finding as a tracking item, not a verified failure.**

**Mitigation (out of scope):** add per-service `paths` filters to railway.toml so docs-only paths (`docs/**`, `*.md`, `STATUS.md`, `ZMN_ROADMAP.md`, `session_outputs/**`) skip redeploy.

**This audit's commit:** if RAILWAY_REDEPLOY_DISCIPLINE_001 is real, this docs commit will trigger redeploys of all 8 services. **Footnote**: that itself becomes a new datapoint.
