# Setting up a new machine for ZMN development

Run these commands once when you pull the repo to a new machine. Takes ~10 minutes
end-to-end if Python and a terminal are already installed, longer if installing
Python from scratch.

Verify install when done with `claude mcp list` — should show 15 servers.

---

## 1. Git tracking already handles

Pulling the repo gives you:

- `.mcp.json` — 15 MCP servers registered (API keys are `${VAR}` placeholders,
  resolve at runtime from env vars)
- `.claude/skills/` — 4 Anthropic skills (mcp-builder, skill-creator,
  frontend-design, webapp-testing)
- `docs/CLAUDE_TOOLING_INVENTORY.md` — usage cheat sheet for what each MCP
  is for and when to reach for it
- `CLAUDE.md` — project-specific CC instructions

What git tracking does NOT handle: CLI tools, local env vars, OAuth state,
Solana CLI. Those are installed per-machine (steps 2-6 below).

---

## 2. Install Python 3.11+ and pipx

**Windows:** install Python from https://www.python.org/downloads/ (NOT the
Microsoft Store stub — it's a redirect that breaks pipx). During install,
tick "Add python.exe to PATH". Then open a fresh terminal:

```bash
python -m pip install --user pipx
python -m pipx ensurepath
# close + reopen terminal so PATH picks up pipx
```

**Mac/Linux:** Python 3.11+ is usually pre-installed. If not, `brew install python@3.11`
(mac) or `apt install python3.11 python3-pip` (Ubuntu/Debian). Then:

```bash
python3 -m pip install --user pipx
python3 -m pipx ensurepath
```

Verify: `pipx --version` returns a version (not "command not found").

---

## 3. Install CLI tools via pipx

All four installs are standalone — if one fails the others still succeed.

```bash
pipx install postgres-mcp
pipx install semgrep
pipx install ruff
pipx install black
pipx install mypy
```

Verify with:

```bash
postgres-mcp --version
semgrep --version
ruff --version
black --version
mypy --version
```

**Do NOT `pipx install python-lft-mcp`** — it's not on PyPI under that name
(3 install attempts failed 2026-04-19). The ruff/black/mypy trio above
covers the same ground.

---

## 4. Install Solana CLI (only if using Jupiter MCP)

Jupiter MCP requires a Solana keypair. A throwaway keypair generated by
`solana-keygen` is the recommended pattern — NEVER fund it, NEVER use it
for production keys.

**Windows** — Developer Mode must be ON for the Solana installer symlinks
(Settings → Privacy & security → For developers → Developer Mode):

```bash
cmd /c "curl https://release.anza.xyz/stable/solana-install-init-x86_64-pc-windows-msvc.exe --output C:\solana-install-tmp\solana-install-init.exe --create-dirs"
C:\solana-install-tmp\solana-install-init.exe stable
# restart terminal so PATH picks up solana-keygen
```

**Mac/Linux:**

```bash
sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

Verify: `solana-keygen --version`

Generate the throwaway key (only needed once Jupiter MCP is registered in
`.mcp.json` — not registered as of 2026-04-19):

```bash
mkdir -p .claude/keys
solana-keygen new --no-bip39-passphrase --force --outfile .claude/keys/jupiter_mcp_throwaway.json
# confirm pubkey has zero balance before any use
```

`.claude/keys/` is gitignored.

## 5. Export env vars for MCP auth

The `${VAR}` placeholders in `.mcp.json` resolve at runtime from your shell
environment. Pull current values from Railway (they already exist there):

```bash
railway login
railway variables -s bot_core --kv | grep -E '^(HELIUS_API_KEY|NANSEN_API_KEY|VYBE_API_KEY|BIRDEYE_API_KEY|DATABASE_URL|REDIS_URL)='
```

Add to shell profile (`~/.bashrc`, `~/.zshrc`, or Windows user env vars):

```bash
export HELIUS_API_KEY=<value>
export NANSEN_API_KEY=<value>
export VYBE_API_KEY=<value>
export BIRDEYE_API_KEY=<value>           # optional
export COINGECKO_API_KEY=<value>         # optional, paid tier only
export DATABASE_URL=<DATABASE_PUBLIC_URL — gondola.proxy.rlwy.net:29062>
export REDIS_URL=<REDIS_PUBLIC_URL>
```

**Never commit these values.** Close the terminal and open a fresh one so the
exports take effect.

## 6. OAuth the remote MCPs (when you have browser focus)

These launch a browser flow on first tool call. Easier to front-load:

```bash

claude mcp login github
claude mcp login gmail    # if already stubbed
claude mcp login gcal     # if already stubbed
```


## 8. First CC session checks

When you spawn your first CC session on this machine, CC reads `CLAUDE.md`
and this file automatically. Ask CC to verify its tooling is online before
real work:

```
> check your MCP list and confirm which ones are green
```

If CC reports anything unexpected, re-run the step it came from rather than
working around it.

## Gotchas

- **`pipx ensurepath` needs a fresh terminal.** Existing terminals won't see
  the PATH update until restart.
- **Windows Python from Microsoft Store is a stub.** If `pipx install ...`
  hangs or fails silently, check `where python` — if it points to
  `Microsoft\WindowsApps`, uninstall the store version and install from
  python.org.
- **Solana-keygen on Windows needs Developer Mode** for symlinks. Without it
  the installer fails with a cryptic error about creating junctions.
- **Do not install `python-lft-mcp`.** Not on PyPI. Use the separate
  ruff/black/mypy CLIs.
- **Never run `--access-mode=unrestricted` on Postgres MCP** outside a
  deliberate, Jay-approved migration session. See the security section of
  `docs/CLAUDE_TOOLING_INVENTORY.md`.
